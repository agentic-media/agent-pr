name: lordship-author-pr
description: >-
  Open a PR on the lordship repo (where /lordship/authors/ lives) with
  a staged author yaml + asset bundle. Reads the staged yaml at
  `/shared/runs/<run>/author-yaml/<slug>.yaml` and any binaries under
  `/shared/runs/<run>/author-assets/<slug>/`, clones the lordship repo
  via `gh`, drops the files into `authors/<slug>.yaml` and
  `authors/avatars/<slug>/...`, commits, pushes a branch like
  `assets/<slug>-<run>`, and opens a PR. Returns the PR URL + branch +
  commit SHA. The PR-agent does NOT merge — operator review is
  intentional.
inputs:
  - name: --slug
    type: 'string — author slug (matches the staged yaml filename)'
  - name: --run
    type: 'string — run id used to build the staged paths and the branch suffix'
  - name: --lordship-repo
    type: 'string — `<owner>/<name>` of the lordship repo (e.g. venetanji/agentic-media-lordship)'
  - name: --staged-yaml-path
    type: 'optional string — override; default `/shared/runs/<run>/author-yaml/<slug>.yaml`'
  - name: --staged-assets-dir
    type: 'optional string — override; default `/shared/runs/<run>/author-assets/<slug>/`'
  - name: --pr-title
    type: 'optional string — override default `Author update: <slug> (<run>)`'
  - name: --base-branch

  - name: --clone-dir
    type: 'optional string — default `/shared/runs/<run>/lordship-repo`'
outputs:
  - name: stdout
    type: 'application/json — { ok, pr_url, branch, commit_sha, files }'

  - hits_external_api: 'github.com (gh CLI)'

# When to use

After `image-generate` + `asset-transcode` have populated
`/shared/runs/<run>/author-assets/<slug>/` with the variants and
`author-yaml-write` has staged
`/shared/runs/<run>/author-yaml/<slug>.yaml`. This skill is the only
sanctioned path for landing changes in `/lordship/authors/` — the
runtime bind is read-only by design (see lordship `authors/_schema.md`).

# How it works

1. Validate inputs:
   - Staged yaml exists and parses as YAML.
   - Staged assets dir exists and contains at least `avatar.webp`.
   - The yaml's `slug` matches `<slug>` arg; mismatch → exit 2.
2. `gh repo clone <lordship-repo> <clone-dir>` (idempotent — reuse
   if already a clone of the same remote).
3. `git fetch origin`, `git checkout -B assets/<slug>-<run>
   origin/<base-branch>`.
4. Copy staged yaml → `<clone-dir>/authors/<slug>.yaml`.
5. Copy staged assets → `<clone-dir>/authors/avatars/<slug>/`,
   preserving the gallery subdirectory layout.
6. `git -C <clone-dir> add authors/<slug>.yaml authors/avatars/<slug>/`.
7. Commit message:
   ```
   Author asset update: <slug>

   <yaml diff summary>
   <asset list>
   ```
8. `git push -u origin <branch>`.
9. `gh pr create -R <lordship-repo> --base <base-branch> --head <branch>
   --title "<pr-title>" --body "<auto-generated>"`. The body lists
   variant dimensions (read from each WebP via Pillow), the
   yaml-diff summary, and a link back to the run dir.
10. Print one JSON object with `pr_url`, `branch`, `commit_sha`, and
    `files`.

# CLI shape

```bash
python /agent/skills/lordship-author-pr/lordship_author_pr.py \
  --slug elena-moretti \
  --run pr-asset-refresh-1777291234 \
  --lordship-repo venetanji/agentic-media-lordship
```

# Failure modes

- Staged yaml missing → exit 2.
- avatar.webp missing → exit 2 (avatar is required; hero/og/gallery
  optional).
- Any non-WebP file in the staged assets dir → exit 2 with the
  offending filename. Transcoding is upstream's job; this skill
  refuses to ship anything that wasn't already converted.
- `gh pr create` fails because a PR for this branch already exists →
  fall through, print the existing PR URL with `ok: true`.

# Notes

- The PR is NOT merged automatically — operator review is the
  contract for `/lordship/authors/` writes.
- After merge, the orchestrator (or operator) runs `author-sync` per
  consumer site to project the new images + JSON into the live sites.
- This skill does NOT generate images, transcode, or write the yaml.
  It only opens the PR. Earlier pipeline stages (image_generate +
  Pillow transcode + author-yaml-write) are the producers; this
  skill is the shipper.
